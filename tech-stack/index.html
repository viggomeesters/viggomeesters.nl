How this stack is curated
Selection rule

A tool belongs here when it changes the default way work is captured, built, verified, automated, or published. Nice-to-have apps and short experiments stay out until they become part of the operating surface.

Proof rule

Each section should name what the tool layer enables, where it sits in the workflow, and what would make the page stale enough to update.

Update trigger

Update the stack when a default device, app, agent, deployment surface, or command bus changes the way work is actually shipped. Do not update it for one-off trials that never become part of the operating loop.

What is intentionally not here
Personal Knowledge System components

Vault structure, schemas, validation, capture routes, skills, and agent interfaces belong under Personal Knowledge System. This page stays focused on practical stack surfaces: hardware, software, agents, developer tools, hosting, and automation.
