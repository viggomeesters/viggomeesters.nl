1Wat je bouwt

Het doel is simpel: een eigen AI-agent die je via Telegram kunt appen. Jij stuurt een bericht. Telegram stuurt dat naar Hermes. Hermes gebruikt een AI-model, voert eventueel tools uit op je pc, en stuurt antwoord terug.

Voor een beginner is de handigste route niet: alles zelf uitzoeken. De handigste route is: eerst Codex installeren, daarna Codex vragen om Hermes + Telegram netjes voor je te installeren.

Jij in Telegram → Hermes Gateway → Hermes Agent → OpenAI via Codex / andere provider

In deze gids betekent “lokaal draaien”: de agent-runtime, tools, scripts, bestanden, Telegram-gateway en sessies draaien op jouw machine. Het AI-model zelf mag gewoon via OpenAI/Codex, Nous, OpenRouter of een andere provider komen.

Belangrijk uitgangspunt

Als je al een ChatGPT-abonnement hebt waarmee je Codex kunt gebruiken, is dat een logische start. Codex ondersteunt inloggen met je ChatGPT-account, en Hermes kan ook met de OpenAI Codex-provider werken. Je hoeft dit dus niet meteen als “API key project” te behandelen.

2Mini-woordenlijst

Als dit nieuw voor je is, zijn vooral de woorden verwarrend. Deze termen kom je tegen:

WSL

Windows Subsystem for Linux. Je draait Ubuntu/Linux binnen Windows, zonder een aparte Linux-pc te installeren.

Ubuntu

Een populaire Linux-versie. In deze gids is Ubuntu de plek waar Codex en Hermes hun terminal-werk doen.

Terminal

Een tekstvenster waarin je commando’s plakt. Denk: PowerShell of Ubuntu, maar dan voor installatie en beheer.

Gateway

Het deel van Hermes dat Telegram-berichten ontvangt en terugstuurt. Zonder gateway moet je Hermes via de terminal gebruiken.

Provider

De partij die het AI-model levert, bijvoorbeeld OpenAI/Codex, Nous of OpenRouter.

Backend

De plek waar je agent duurzame data kwijt kan: notities, taken, logs, context, kennis of geheugen.

Je ziet sommige woorden ook als hover-uitleg in de tekst, bijvoorbeeld WSL, provider en gateway.

3Start slimmer: laat Codex de installatie doen

De beste noob-route is niet om deze gids perfect handmatig te volgen. Installeer eerst Codex CLI, log in met je ChatGPT-account, en laat Codex daarna de rest uitvoeren terwijl jij meekijkt.

Waarom? Omdat installaties vaak vastlopen op kleine machine-specifieke dingen: ontbrekende packages, PATH-problemen, een oude Windows-build, een terminal die anders werkt, of een service die niet start. Een agent op dezelfde pc kan die foutmeldingen lezen en meteen de volgende stap proberen.

Prompt voor Codex

Plak na het installeren van Codex ongeveer dit:

Ik wil Hermes Agent installeren op deze oude Windows-pc via WSL/Ubuntu en koppelen aan Telegram.
Gebruik mijn ChatGPT/Codex-login waar mogelijk voor OpenAI/Codex-auth.
Installeer Hermes, run hermes doctor, configureer Telegram Gateway, zet veilige allowlist/pairing aan,
adviseer een geheugen/backend setup met Holographic Memory + jsonl-vault of Notion,
en stop voordat je destructieve of publieke acties uitvoert.

Blijf wel opletten: Codex mag helpen, maar jij moet tokens, accounts en security-keuzes bewust behandelen.

5WSL en Ubuntu installeren

Open PowerShell als administrator. PowerShell is de Windows-terminal. Plak dit commando:

wsl --install -d Ubuntu

Herstart Windows als daarom gevraagd wordt. Open daarna Ubuntu vanuit het Startmenu. Je zit nu in Linux-binnen-Windows.

Update Ubuntu en installeer basisgereedschap:

sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl xz-utils

Wat betekent dit?

  • sudo: voer uit als beheerder binnen Linux.
  • apt: de pakketmanager van Ubuntu.
  • git: haalt code uit GitHub-repositories.
  • curl: downloadt scripts of webdata via de terminal.
  • xz-utils: nodig voor sommige gecomprimeerde downloads.

Check dat je echt in Linux zit:

uname -a

6Codex installeren

Codex is de command-line coding agent van OpenAI. Hij draait lokaal op je computer en kan helpen om Hermes te installeren, foutmeldingen te lezen en configuratiebestanden aan te passen.

In Ubuntu/WSL kun je de officiële installer gebruiken:

curl -fsSL https://chatgpt.com/codex/install.sh | sh

Start daarna Codex:

codex

Kies Sign in with ChatGPT. Daarmee gebruik je Codex via je ChatGPT-account/abonnement. Dit is juist de route die ik voor beginners zou kiezen.

Alternatief kan Codex ook via npm: npm install -g @openai/codex. Gebruik de installer hierboven als default.

7Hermes installeren

Je kunt dit handmatig doen, maar als Codex al draait: laat Codex meekijken en uitvoeren. Handmatig is dit de basis:

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
source ~/.bashrc
hermes doctor

hermes doctor is de gezondheidscheck. Die vertelt wat ontbreekt of verkeerd staat.

Als hermes niet gevonden wordt, staat de launcher waarschijnlijk nog niet in je PATH. Plak dan:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

Test daarna:

hermes chat -q "Geef één zin terug: Hermes werkt."

8OpenAI/Codex login instellen

Hermes heeft een modelprovider nodig: dat is de plek waar het taalmodel vandaan komt. Voor deze route is OpenAI/Codex logisch, omdat je Codex al met je ChatGPT-account gebruikt.

Probeer eerst de login-route voor OpenAI Codex:

hermes login --provider openai-codex

Daarna kun je je model/provider kiezen:

hermes model

Algemene setup kan ook:

hermes setup

Als je juist Nous Portal wilt gebruiken, kan dat ook:

hermes setup --portal

Maar voor iemand met een werkende Codex/ChatGPT-route zou ik beginnen met Codex/OpenAI. Minder losse API-key-rommel, sneller naar een werkende agent.

9Telegram koppelen

Telegram werkt via een bot. Een bot is een speciaal Telegram-account dat berichten kan ontvangen en beantwoorden via een token.

  1. Open Telegram.
  2. Zoek @BotFather.
  3. Stuur /newbot.
  4. Kies een naam, bijvoorbeeld Remko Hermes.
  5. Kies een username die eindigt op bot, bijvoorbeeld remko_hermes_bot.
  6. BotFather geeft een token. Dat is een soort wachtwoord voor je bot. Deel die niet.

Configureer daarna Hermes Gateway:

hermes gateway setup

Kies Telegram en plak de BotFather-token. Start de gateway om te testen:

hermes gateway run

Stuur nu een DM naar je bot. Als hij antwoordt, werkt de basis.

10Geheugen en backend toevoegen

Een kale Telegram-agent is leuk, maar beperkt. De echte waarde ontstaat als Hermes dingen kan onthouden en ergens duurzaam kan opslaan.

Ik zou twee lagen toevoegen:

LaagWat doet het?Mijn advies
GeheugenFeiten, voorkeuren, terugkerende context en relaties onthouden.Installeer/activeer Holographic Memory als die beschikbaar is in je Hermes setup.
BackendDuurzame opslag voor notities, taken, logs, documenten, captures of kennis.Begin simpel met jsonl-vault. Kies Notion als je liever een bekende UI hebt.

Wat is het verschil?

  • Holographic Memory: goed voor “wie is wie?”, voorkeuren, terugkerende context en verbanden tussen dingen.
  • jsonl-vault: simpele bestanden met één JSON-record per regel. Saai, controleerbaar en goed voor agents.
  • Notion: prettig als mensen zelf willen meekijken en klikken, maar minder Git/native dan bestanden.
  • Obsidian/Markdown: sterk als je zelf een lokale kennisbank wilt beheren.
Mijn startadvies

Voor een persoonlijke agent: Holographic Memory + jsonl-vault. Voor iemand die vooral visueel wil beheren: Holographic Memory + Notion. Laat Codex de beschikbare Hermes memory/backend-opties inspecteren en vraag om de kleinste werkende setup.

Prompt voor Codex:

Controleer welke memory/backends deze Hermes-installatie ondersteunt.
Adviseer en configureer een beginner-vriendelijke setup met Holographic Memory plus jsonl-vault of Notion.
Gebruik geen geheime tokens in chatoutput. Toon alleen welke integraties werken en hoe ik ze test.

11Permanent laten draaien

Voor een eerste test is foreground draaien genoeg:

hermes gateway run

Dat betekent: zolang dit terminalvenster open is, luistert Hermes naar Telegram.

Wil je hem als achtergrondservice draaien, check dan eerst of systemd in WSL werkt:

systemctl --user status

Als dat werkt:

hermes gateway install
hermes gateway start
hermes gateway status

Als Windows gaat slapen, stopt je “server”. Zet sleep uit, laat de machine aan staan, of verhuis naar een mini-pc/VPS.

12Veiligheid: zet geen terminal-agent publiek open

Een Telegram-agent met terminal-tools is krachtig. Hij kan, afhankelijk van je instellingen, commando’s uitvoeren, bestanden lezen, browseracties doen en externe services raken.

  • Deel de bot niet breed.
  • Zet hem niet open voor iedereen.
  • Gebruik allowlist of pairing.
  • Zet geen API keys of tokens in chatberichten.
  • Laat betaling, productie, publicatie en destructieve acties bewust gated.

Als een onbekende user de bot DM’t, kan Hermes met pairing codes werken. Je kunt aanvragen zien en goedkeuren met:

hermes pairing list
hermes pairing approve telegram CODE

Gebruik GATEWAY_ALLOW_ALL_USERS=true alleen als je bewust een publieke speeltuin bouwt. Voor een persoonlijke agent is dat bijna nooit de juiste default.

13Succescheck

Je bent klaar met de eerste setup als deze commando’s werken:

codex --version
hermes doctor
hermes chat -q "Geef één zin terug: Hermes werkt."
hermes gateway status

En als dit in Telegram werkt:

/status
/help
/new

Daarna is de beste volgende stap niet meer installeren, maar gebruiken. Geef hem echte kleine opdrachten: vat een link samen, schrijf een mailconcept, zoek iets uit, maak een checklist, of laat hem een bestand analyseren.

De nuchtere eindconclusie

Een oude Windows-pc met WSL is prima als startbak. Maar maak het slimmer: installeer Codex, laat Codex Hermes installeren, koppel Telegram, voeg geheugen en een backend toe, en kijk dan of je het zeven dagen echt gebruikt. Daarna pas beslissen of je naar mini-pc of VPS verhuist.
