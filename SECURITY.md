# Security Policy

## Supported Versions

Security updates are provided for the latest released version unless stated
otherwise in the release notes.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability. Contact the maintainer
privately through [GitHub's private vulnerability reporting form](https://github.com/viggomeesters/viggomeesters.nl/security/advisories/new) with:

- affected version or commit;
- reproduction steps;
- expected impact;
- any safe proof-of-concept details.

## Public Repository Hygiene

This repository must not contain secrets, personal data, customer data, private
screenshots, `.env` files, access tokens, private keys, or proprietary source
material.
