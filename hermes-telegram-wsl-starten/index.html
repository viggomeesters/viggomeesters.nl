1Wat je bouwt

Het doel is simpel: een eigen AI-agent die je via Telegram kunt appen, terwijl de agent op een oude Windows-pc, mini-pc of VPS draait.

De agent is Hermes Agent. De chatinterface is Telegram. De oude Windows-pc draait bij voorkeur WSL2 met Ubuntu, zodat Hermes in een normale Linux-omgeving kan draaien.

Telegram → Hermes Gateway → Hermes Agent → OpenAI / Nous / andere provider

Daarna kun je hem vragen stellen, taken laten uitvoeren, webinformatie laten ophalen, bestanden laten analyseren, code laten aanpassen, samenvattingen laten maken of kleine automatiseringen laten draaien.

2Belangrijke nuance: lokaal draaien betekent niet lokaal model

“Ik draai thuis een lokale AI” klinkt alsof het hele taalmodel op je eigen pc draait. Dat kan, maar dat is meestal niet de beste eerste stap.

In deze setup draait vooral de agent-runtime lokaal: Hermes, de tools, de Telegram-gateway, je bestanden, je scripts en je sessies. Het brein komt meestal van een modelprovider zoals OpenAI, Nous Portal, OpenRouter, Anthropic of een andere OpenAI-compatible endpoint.

ChatGPT abonnement is niet altijd OpenAI API

Een ChatGPT Plus/Pro abonnement is niet automatisch hetzelfde als OpenAI API-tegoed. Voor direct OpenAI API-gebruik heb je meestal een API key en API-billing nodig. Als je provider-gedoe wilt vermijden, is Nous Portal vaak de makkelijkste start omdat modeltoegang en toolgateway daar in één setup zitten.

Mijn advies

Start op de oude Windows-pc met WSL. Krijg Hermes + Telegram werkend. Gebruik het een week. Als je de bot echt dagelijks gebruikt, verhuis hem daarna naar een mini-pc met Ubuntu of een VPS.

4WSL en Ubuntu installeren

Open PowerShell als administrator en installeer Ubuntu via WSL:

wsl --install -d Ubuntu

Herstart Windows als daarom gevraagd wordt. Open daarna Ubuntu vanuit het Startmenu en update de basispakketten:

sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl xz-utils

Check dat je echt in Linux zit:

uname -a

5Hermes installeren

Installeer Hermes in Ubuntu/WSL:

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
source ~/.bashrc
hermes doctor

Als het commando hermes niet gevonden wordt, voeg dan de lokale bin-map toe aan je shell:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

Test daarna of Hermes start:

hermes chat -q "Geef één zin terug: Hermes werkt."

6Modelprovider instellen

De makkelijkste route is de interactieve setup:

hermes setup

Als je Nous Portal gebruikt, is dit de kortste route:

hermes setup --portal

Als je OpenAI wilt gebruiken, kies je via de modelconfiguratie je provider en model:

hermes model

Gebruik daarna opnieuw:

hermes doctor

Los eerst de meldingen op voordat je Telegram koppelt. Dat scheelt zoekwerk: als de CLI al niet werkt, gaat de Telegram-bot ook niet magisch werken.

7Telegram koppelen

Maak eerst een Telegram-bot aan via @BotFather:

  1. Open Telegram.
  2. Zoek @BotFather.
  3. Stuur /newbot.
  4. Kies een naam, bijvoorbeeld Remko Hermes.
  5. Kies een username die eindigt op bot, bijvoorbeeld remko_hermes_bot.
  6. Bewaar de token die BotFather geeft. Deel die niet.

Configureer daarna de Hermes gateway:

hermes gateway setup

Kies Telegram en plak de BotFather-token. Start daarna de gateway in foreground:

hermes gateway run

Stuur nu een DM naar je Telegram-bot. Als hij reageert, is de basis klaar.

8Permanent laten draaien

Voor een eerste test is dit genoeg:

hermes gateway run

Laat de Ubuntu-terminal open staan. Niet mooi, wel eerlijk: dit bewijst eerst of je de agent gebruikt.

Als je hem als service wilt draaien, check dan of systemd in WSL werkt:

systemctl --user status

Als dat werkt:

hermes gateway install
hermes gateway start
hermes gateway status

Als WSL afsluit zodra niemand is ingelogd of Windows gaat slapen, is dit geen Hermes-probleem maar een serverprobleem. Zet sleep uit, laat de machine aan staan, of verhuis naar een mini-pc/VPS.

9Veiligheid: zet geen terminal-agent publiek open

Een Telegram-bot met agent-tools is geen speelgoedbot. Hij kan, afhankelijk van je toolconfiguratie, commando’s uitvoeren, bestanden lezen, browseracties doen en externe services raken.

  • Deel de bot niet breed.
  • Zet hem niet open voor iedereen.
  • Gebruik allowlist of pairing.
  • Zet geen API keys in chatberichten.
  • Laat betalings-, productie- of destructieve acties altijd bewust gated.

Als een onbekende user de bot DM’t, kan Hermes met pairing codes werken. Je kunt aanvragen zien en goedkeuren met:

hermes pairing list
hermes pairing approve telegram CODE

Gebruik GATEWAY_ALLOW_ALL_USERS=true alleen als je bewust een publieke speeltuin bouwt. Voor een persoonlijke agent is dat vrijwel nooit de juiste default.

10Succescheck

Je bent klaar met de eerste setup als deze commando’s werken:

hermes doctor
hermes chat -q "Geef één zin terug: Hermes werkt."
hermes gateway status

En als dit in Telegram werkt:

/status
/help
/new

Daarna is de beste volgende stap niet meer installeren, maar gebruiken. Geef hem echte kleine opdrachten: vat een link samen, schrijf een mailconcept, zoek iets uit, maak een checklist, of laat hem een bestand analyseren.

De nuchtere eindconclusie

Een oude Windows-pc met WSL is prima als startbak. Noem het alleen nog geen server voordat hij zeven dagen betrouwbaar bereikbaar is geweest. Als je hem dagelijks gaat gebruiken, is een mini-pc met Ubuntu of een VPS de volwassen stap.
